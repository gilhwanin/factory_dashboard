# 업체별 발주량 갱신 로직

## 전체 흐름

```
[발주량 갱신 버튼 클릭]
  → widget.py:119  →  data_writer.py:642 on_click_update_order_qty_after()
  → 업체별 발주량 계산 (logic/cal_values.py)
  → ORDER_DASHBOARD 테이블 UPDATE
  → 원료/소스/야채 테이블 재계산
  → UI 새로고침
```

---

## 1. PACSU(팩수) 조회 (공통)

모든 업체 공통으로, 발주량 계산 전에 팩 사이즈를 조회한다.

- **DB:** `GFOOD_B`
- **테이블:** `MASTER`
- **쿼리:**
  ```sql
  SELECT TOP 1 PACSU FROM MASTER WHERE CO = %s
  ```

---

## 2. 업체별 발주량 조회 쿼리

각 상품의 `vendor`(업체)에 따라 **다른 DB·테이블·쿼리**를 사용한다.

### 홈플러스

| 항목 | 값 |
|---|---|
| **DB** | `GWCHUL` |
| **테이블** | `PAN` |
| **쿼리** | `SELECT ISNULL(SUM(PAN), 0) AS sum_pan FROM PAN WHERE CO = %s AND CONVERT(DATE, PDATE) = %s AND DE = 'N'` |
| **계산** | 조회값 × PACSU |

### 이마트

| 항목 | 값 |
|---|---|
| **DB** | `GFOOD_B` |
| **테이블** | `MMASTER` → `MPAN` |
| **쿼리 1** | `SELECT TOP 1 CO FROM MMASTER WHERE TCO = %s` |
| **쿼리 2** | `SELECT SUM(PANKG) AS sum_pan FROM MPAN WHERE CO = %s AND CONVERT(DATE, SDATE) = %s AND DE = 'N'` |
| **계산** | 조회값 × PACSU |

### 마켓컬리

| 항목 | 값 |
|---|---|
| **DB** | `GFOOD_B` |
| **테이블** | `MPAN` |
| **쿼리** | `SELECT SUM(PANKG) AS sum_pan FROM MPAN WHERE CO = %s AND CONVERT(DATE, SDATE) = %s AND DE = 'N'` |
| **비고** | 토요일인 경우 다음날(일요일)로 날짜 보정 |
| **계산** | 조회값 × PACSU |

### 롯데

| 항목 | 값 |
|---|---|
| **DB** | `GFOOD_B` |
| **테이블** | `MJEN` → `MPAN` |
| **쿼리 1** | `SELECT JNO FROM MJEN WHERE rname LIKE '%롯데%' AND JBIGO = '양념육' AND CONVERT(DATE, SDATE) = %s AND DE = 'N'` |
| **쿼리 2** | `SELECT SUM(PANKG) AS sum_pan FROM MPAN WHERE JNO IN (...) AND CO = %s AND DE = 'N'` |
| **계산** | 조회값 × PACSU |

### 코스온

| 항목 | 값 |
|---|---|
| **DB** | `GWCHUL` |
| **테이블** | `MASTER` → `COSONC` |
| **쿼리 1** | `SELECT TOP 1 TCO3 FROM MASTER WHERE CO = %s` |
| **쿼리 2** | `SELECT TOP 1 FINAL_QTY FROM COSONC WHERE LCODE = %s AND CONVERT(DATE, LDATE) = %s` |

### 코스트코

| 항목 | 값 |
|---|---|
| **DB** | `GWCHUL` |
| **테이블** | `COS_B` |
| **쿼리** | `SELECT ISNULL(SUM(CONVERT(int, C17)), 0) AS sum_pack FROM COS_B WHERE REPLACE(RTRIM(LTRIM(C29)), ' ', '') = %s AND CONVERT(DATE, C06) = %s` |
| **비고** | 날짜 보정 로직 있음 (day_adj, mon_adj 파라미터). "자율" 타입은 `round(total_pack / pack_weight)`, "일반" 타입은 total_pack 그대로 사용 |

## 3. 결과 저장

- **DB:** `GP`
- **테이블:** `ORDER_DASHBOARD`
- **쿼리:**
  ```sql
  UPDATE ORDER_DASHBOARD
  SET order_qty_after = %s
  WHERE CONVERT(DATE, sdate) = %s AND co = %s
  ```

---

## 4. 후속 재계산

UPDATE 후 `GFOOD_B.RECIPE` 테이블을 참조하여 관련 3개 테이블을 연쇄 재계산한다.

| 함수 | 대상 테이블 | 내용 |
|---|---|---|
| `recalc_dashboard_raw_keep_manual()` | `DASHBOARD_RAW` | 원료 소요량 재계산 (레시피 키워드: "(정선)") |
| `recalc_dashboard_sauce_keep_manual()` | `DASHBOARD_SAUCE` | 소스 소요량 재계산 (레시피 키워드: "소스") |
| `recalc_dashboard_vege_keep_manual()` | `DASHBOARD_VEGE` | 야채 소요량 재계산 (특정 BCO 필터) |

---

## 5. DB 연결 정보 요약

| DB명 | 역할 | 주요 테이블 |
|---|---|---|
| **GP** | 메인 대시보드 | `ORDER_DASHBOARD`, `DASHBOARD_RAW`, `DASHBOARD_SAUCE`, `DASHBOARD_VEGE` |
| **GFOOD_B** | 마스터·레시피 정보 | `MASTER`, `MMASTER`, `MPAN`, `MJEN`, `RECIPE` |
| **GWCHUL** | 출하·발주 데이터 | `PAN`, `COS_B`, `COSONC` |

- 모두 `pymssql`로 MSSQL Server에 접속
- 서버 정보는 암호화된 ENV에서 로드
